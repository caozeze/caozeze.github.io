Improving Community Retention: An Investigation into Retention Enhancement in a CSDN-like Content Community
Mar 3, 2022

1. JD Consumer Behaviour Report

1.1. Project Background

Understanding user behaviour on digital platforms is crucial for enhancing user experience and driving business growth. This project aims to provide an in-depth analysis of user behaviour on a digital platform, revealing patterns and preferences in user actions such as browsing, purchasing, sharing, and liking.

Project Link: How to Improve Community Retention: An Exploration of Retention Enhancement in a CSDN-like Content Community
Project Timeline: 12/02/2022 - 03/03/2022
Role: Data Operations Analyst / Project Management
Tools Used: Python (Pandas, Numpy, Pyecharts, Seaborn, etc.)
Source Data: The dataset for this project comprises two parts: one part is the actual purchase order data generated by users, and the other part is user preference data collected through daily operations.
Responsibilities: Project Management / Idea Design / Article Production / Data Processing and Analysis with Pandas / Visualisation with Pyecharts
Data Source: The dataset for this project comprises user behaviour data collected from the platform, including actions such as viewing, buying, sharing, liking, rewarding, and participating in activities.

1.2. Project Process

  1. Conducted data processing with Pandas, managing null values, missing values, duplicates, and data formatting
  2. Established a research outline for the visualisation of retention and activity rates across different dimensions
  3. Implemented user segmentation based on the Recency, Frequency, Monetary (RFM) model for precision operations
  4. Conducted comparative analysis of user behaviour (cohort analysis) and behaviour correlation analysis

1.3. Project Value

  1. Identified key differential behaviours (Aha-Moment) between low-retention and high-retention users
  2. Applied the RFM model for precision user operations and researched key indicators for improving retention rates
  3. Provided core recommendations for platform operations, such as new user onboarding

2. Detailed Report

2.1. Analysis Objectives

Our data analysis objectives mainly include the following aspects:

  1. Determine the frequency of different user actions on the platform and identify any trends or patterns.
  2. Identify any anomalies in the frequency of user actions and investigate potential causes.
  3. Analyse the correlation between different user actions.
  4. Identify key user actions that influence other user behaviours.

2.2. Analysis Results and Suggestions

Our data analysis reveals the following key insights:

  1. The frequency of user actions on the platform (such as browsing, buying, sharing, liking) shows certain trends. For example, the frequency of user browsing behaviour has the highest similarity with the PV/UV graph of the day/month. There is an anomaly in the frequency of user collection behaviour in mid-August, but no explanatory factors can be proposed at present. The frequency of user rewarding behaviour has a peak in November, which may be related to the actual appearance of popular articles/resources in November. User activity behaviour has certain peaks in July and December, but the specific effects need further observation.
  2. The analysis raises some questions and speculations, such as: Does the behaviour object ID (target_id) contain other objects besides article objects (e.g., user objects, activity objects, product objects, etc.)? If a user buys/likes/shares/collects/participates in an activity, does the user need to view this page first? If a user gives a reward, what does the behaviour object ID (target_id) point to? Is it another user or an article?
  3. The analysis provides answers to the above questions. For example, if a user buys/likes/shares/collects, the user needs to view this page first. If a user gives a reward, the user definitely does not reward another user, nor a shareable and purchasable object. It may be a column or resource file (reward to download). If a user shares (Share), the user just shares the content (the first major class object).
  4. The analysis also conducted a correlation analysis of various indicators, finding that whether it is high-quality users or overall users, they have a high degree of association in the following pairs of behaviours: view and buy, view and share, reward and share, reward and comp. Looking at the points where the difference in behaviour correlation between high-quality users and overall users is high, the correlation between vote and other behaviours has a large increase (except for buy). This may indicate that vote may be a key factor influencing other user behaviours.

2.3. Conclusion

Although the content of this analysis needs further improvement, it has already provided us with some important insights into user behaviour on the digital platform. We will continue to improve our analysis methods to provide a deeper and more comprehensive understanding.
